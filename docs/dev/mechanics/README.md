# TEP 0.4.0 Mechanics

This directory expands `docs/dev/TEP_0_4_0_MECHANICS_REQUIREMENTS.md` into
implementation-oriented mechanics.

Each file answers:

- what the mechanic is responsible for
- which records and runtime artifacts it owns
- what the agent is allowed to do
- what the API or hooks must enforce
- where the mechanic connects to other mechanics
- which gaps must be closed before 0.4.0
